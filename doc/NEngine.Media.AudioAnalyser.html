    NEngine.Media. AudioAnalyser

    new AudioAnalyser(opts, updatesopt)

    Wrapper for Web Audio API AnalyserNode, adds some basic processing to normalize and clean the raw data of an analyser for easy visualization. To activate a processing filter on the update function, the processing type needs to be included in the instance "updates" array. Then you can access the results from corresponding entries in the "data" object .
    Properties:
    Properties
    Name Type Description
    ctx AudioContext The Web Audio AudioContext
    node AnalyserNode The wrapped node
    updates Array.<String> The list of processors to update on the "update" method. Each entry is maped to the update_"entry_name" method in AudioAnalyser.prototype and executed in the same order.
    Default value: ['raw','normalize', 'usable_log']
    data Object Holds results of processings on the node buffer
    Name Type Description
    raw UInt8Array Data extracted using
    normalize Array.<Number> The "normalize" filter main results.
    normalizeFrecUnit Number The normalize filter dividing factor.
    normalizeFrecPow Number The normalize exponent.
    usableLog Array.<Number> The update_usable_log results.
    usableLogMean Number The mean of all the usableLog values
